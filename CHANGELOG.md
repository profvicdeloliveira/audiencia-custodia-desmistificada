## [1.0.0] - 2025-10-02

### Added
- Publicação inicial do artigo "Audiência de Custódia desmistificada" no repositório acadêmico.
- Inclusão de metadados originais do artigo, preservando informações como autores, data de publicação original, DOI (se aplicável) e referências bibliográficas.
- Transição completa do conteúdo hospedado anteriormente na plataforma Sindepol para o repositório GitHub, garantindo acessibilidade e versionamento contínuo.
- Estruturação do repositório com arquivos README.md, LICENSE e este CHANGELOG.md para conformidade com padrões acadêmicos e de código aberto.
- Adição de instruções para contribuidores, incluindo diretrizes para submissão de pull requests e relatórios de issues relacionados ao artigo.

### Changed
- Migração do formato de publicação de Sindepol para GitHub, adaptando o layout e a formatação para compatibilidade com Markdown e ferramentas de versionamento.
- Atualização de referências internas para refletir a nova localização do repositório, assegurando links funcionais e citações precisas.

### Deprecated
- Nenhuma funcionalidade ou seção depreciada nesta versão inicial.

### Removed
- Remoção de dependências ou elementos específicos da plataforma Sindepol que não são compatíveis com o ambiente GitHub, como plugins proprietários ou formatos de arquivo não-padrão.

### Fixed
- Correção de erros tipográficos e de formatação identificados durante a transição, assegurando integridade acadêmica do conteúdo.
- Resolução de problemas de compatibilidade de arquivos, garantindo que o artigo seja renderizado corretamente em visualizadores Markdown.

### Security
- Implementação de medidas básicas de segurança no repositório, como proteção de branches principais e revisão de código para prevenir introdução de vulnerabilidades.

## Instruções para Contribuidores

Para contribuir com o repositório acadêmico do artigo "Audiência de Custódia desmistificada", siga as diretrizes abaixo:

1. **Fork e Clone**: Faça um fork do repositório e clone-o localmente para trabalhar em suas modificações.
2. **Branching**: Crie uma branch específica para suas alterações (e.g., `feature/nova-secao` ou `fix/erro-tipografico`).
3. **Commits**: Use mensagens de commit claras e descritivas, seguindo convenções como [Conventional Commits](https://www.conventionalcommits.org/).
4. **Pull Requests**: Submeta um pull request com uma descrição detalhada das mudanças, incluindo justificativa acadêmica ou técnica. Aguarde revisão pelos mantenedores.
5. **Issues**: Relate bugs, sugestões ou discussões através de issues no GitHub, utilizando templates apropriados.
6. **Licença e Ética**: Assegure-se de que todas as contribuições respeitam a licença do projeto (e.g., CC BY-SA) e princípios éticos acadêmicos, evitando plágio ou conteúdo não autorizado.
7. **Versionamento**: Mudanças significativas devem ser refletidas neste CHANGELOG.md, seguindo o formato Keep a Changelog.

Contribuições são bem-vindas para aprimorar a disseminação e a qualidade do artigo. Para dúvidas, entre em contato com os mantenedores via issues.
